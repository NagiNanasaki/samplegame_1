## 今日やったこと（2026-02-28）

### 1. scene1.ks → title.ks でシステムボタンが残る問題（済み）
- **問題**: scene1終了後にタイトルへ戻るとSAVEなどのシステムボタンが表示されたまま
- **修正**: scene1.ks 末尾の `[layopt layer="message0" visible="false"]` の直後に `[clearfix name="role_button"]` を追加
- **教訓**: `[mask]` で黒画面の状態でも fixlayer は生きている。jump 前に明示的に `[clearfix]` すること

### 2. Config/Extraからタイトルに戻るとキャラ・ボタンが復帰しない問題（済み）
- **根本原因**: `[awakegame]` がTyranoScriptのレイヤーを復元する際に、iscriptで作成した `#title_ui` DOM要素を消してしまう
- **誤った修正**: プラグインフォルダ（`data/others/plugin/thema_nagi_1/config.ks`）を編集していたが、実際に使われていたのは `data/scenario/config.ks` と `data/scenario/cg.ks` だった
- **正しい修正**:
  1. `data/scenario/config.ks` `*backtitle`: `[awakegame]` → `[jump storage="title.ks"]`
  2. `data/scenario/cg.ks` `*backtitle`: `[awakegame]` → `[clearstack]` + `[jump storage="title.ks"]`
  3. `title.ks` Config/Extra ボタンハンドラ: `sleepgame` → `jump`
- **教訓**:
  - **`[awakegame]` はiscriptで追加したDOM要素を消す**。iscript製のUIと sleepgame/awakegame は相性が悪い
  - `[jump]` で戻る方式に変えると title.ks が頭から再実行される → アニメーション再生・BGM再起動の副作用が出る（↓で対処）
  - TSのシナリオフォルダ（`data/scenario/`）と プラグインフォルダ（`data/others/plugin/`）に同名ファイルが存在することがある。どちらが実際に呼ばれているか必ず確認する

### 3. Config/Extra復帰時にタイトル演出が再生される・BGMが再起動する問題（済み）
- **問題**: `[jump storage="title.ks"]` で戻るため title.ks が頭から再実行 → アニメーションフル再生・`[playbgm]` が再実行
- **修正**:
  - Config/Extra ボタンハンドラで `tf._title_skip_anim = true` をセットしてからジャンプ
  - title.ks のiscript先頭で `var _skipAnim = !!(tf._title_skip_anim); tf._title_skip_anim = false;` で読み取り・リセット
  - `_skipAnim` が true のとき: 霞・キャラ・ロゴ・ボタン・orb を即座に最終状態に設定（animate不使用）、fade_overlay のみ短縮（600ms）で表示
  - `[playbgm]` を `[if exp="!tf._title_skip_anim"]...[endif]` で囲んで復帰時はスキップ
- **教訓**:
  - `tf` 変数は `[jump]` をまたいで保持される（`loadGameData` が呼ばれない限りクリアされない）
  - `[playbgm]` の時点では `tf._title_skip_anim` はまだ `true`（リセットは後の iscript）なので `[if]` で直接参照できる
